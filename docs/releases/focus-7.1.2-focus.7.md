# BliPai Focus v7.1.2-focus.7

发布日期：2026-03-24

## 更新摘要
- 本次继续优化首页“关注”的首批展示、下拉刷新耗时，以及刷新后偶发不继续分页的问题。

## 首批 `16` 条展示
- 首页“关注”现在从首轮开始就固定按 `16` 条展示，不再出现首屏一下子摊出几十条、后续才改成 `16 / 16` 追加的割裂体验。
- 下拉刷新也会立即回到首批 `16` 条展示窗口，保持刷新前后节奏一致。

## 更快的下拉刷新
- 下拉刷新时，会先基于当前池内可见缓存直接重排并优先展示首批 `16` 条，再继续补最新网络结果。
- 刷新等待目标改成“补足当前首批展示所需”，不再继续傻等 `16` 个全新条目才结束。
- 首页“关注”首轮并发也从 `8` 路提高到 `16` 路，减少多轮补抓才能看到完整首批的情况。

## 分页稳定性
- `loadMore` 现在不再只依赖上一次 UI 写入的 `hasMore` 标记；只要缓存里还有没展示的内容，或者底层抓取游标仍可继续，就会继续加载下一批。
- 针对刷新后偶发“已拉到底却不继续加载”的问题，当前补了 FOLLOW 专用的分页兜底逻辑。
