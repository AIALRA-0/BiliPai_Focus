# BliPai Focus v7.1.3-focus.1

发布日期：2026-03-24

## 更新摘要
- 本次同步上游 `BiliPai v7.1.3`，并继续收口首页“关注”的首批展示、刷新回顶，以及刷新后自动补批过早触发的问题。

## 上游 `7.1.3` 同步
- 合并空间页搜索、动态页顶部 Tab 持久化、播放器封面/手动起播、后台播放音频焦点、首页头像分流等上游更新。
- Focus 版继续保留独立包名、与官方版共存安装，以及只接收 Focus 仓库正式 `release` APK 的应用内更新通道。

## 首页“关注”首批 `16` 条
- 首页“关注”现在把“展示窗口”作为唯一真相：首次进入和下拉刷新完成后，都只展示首批 `16` 条可见视频。
- 即使后台缓存池、过滤池或抓取池里已经有更多结果，也不会再提前把几十条一次性摊到首屏。

## 刷新完成后统一回顶
- 过滤后的 FOLLOW 下拉刷新完成后，会先提交新的首批 `16` 条，再统一滚动到顶部。
- 不再沿用刷新前的旧锚点停在原地，减少刷新后的 UI 抽搐和顺序错觉。

## 刷新后的自动补批门闩
- FOLLOW 现在会在“刷新完成但顶部复位尚未处理”期间临时锁住自动 `loadMore`。
- 只有回顶处理完成后，才重新允许底部自动补下一组 `16` 条，避免刷新后一落到底就立刻偷偷加载下一批。
